# 资料搜集流程 SOP（Codex 协作版）

这份 SOP 用来把“全平台搜资料”流程化。目标不是多收藏，而是持续产出可验证、可复用、能指导行动的高质量资料库。

## 1. Codex 能帮你做什么

Codex 可以承担这些环节：

| 环节 | Codex 可做 | 你需要做 |
|---|---|---|
| 选题拆解 | 把一个领域拆成规则、入门、案例、工具、数据、避坑、实操 SOP | 确认你真正要解决的问题 |
| 搜索词设计 | 生成各平台关键词、长尾词、反向避坑词、竞品词 | 提供你已知的行业黑话 |
| 资料采集 | 搜公开网页、读取你保存到项目里的 Markdown/截图/转写稿/链接 | 对需要登录的 App 内容，手动保存链接、截图或文字 |
| 质量筛选 | 按评分表过滤标题党、卖课文、过时内容、伪案例 | 对少数关键判断做人工确认 |
| 信息抽取 | 提炼观点、证据、操作步骤、指标、风险、可测试动作 | 判断是否符合你的资源和能力 |
| 交叉验证 | 对同一结论找多个来源互证，标记矛盾点 | 对平台内真实数据做抽样核验 |
| 资料库整理 | 生成主题索引、学习路线、行动清单、复盘表 | 决定下一步先测试哪个方向 |

重要边界：

- 小红书、抖音精选 App、B站、公众号等平台中，凡是需要登录、App 内浏览、反爬限制或非公开内容的，Codex 不能稳定自动抓取全部内容。
- 但你可以把链接、截图、复制文本、视频转写稿、收藏导出结果放进项目，Codex 可以继续清洗、评分、归类、交叉验证。
- 对“最新规则、平台政策、价格、工具可用性”这类可能变化的信息，Codex 应重新联网核验，不只依赖旧资料。

## 2. 高质量资料判断标准

每条资料按 100 分筛选，低于 60 分不进入主资料库，只放到“待验证/噪音池”。

| 维度 | 分值 | 判断标准 |
|---|---:|---|
| 来源可信度 | 20 | 官方、真实卖家复盘、可追溯作者、公开数据、研究报告优先 |
| 证据密度 | 20 | 有截图、数据、成交记录、样本、实验过程，而不是只喊口号 |
| 可操作性 | 20 | 能落到步骤、指标、模板、工具、检查项 |
| 时效性 | 15 | 平台规则、流量机制、工具教程越新越好；老资料需标注年份 |
| 交叉验证 | 15 | 至少能被 2 个独立来源支持，或能被平台数据验证 |
| 风险意识 | 10 | 明确违规、售后、成本、失败条件，而不是只讲收益 |

一票否决：

- 强诱导卖课，但没有公开有效信息。
- 只展示收益截图，不展示过程、成本、失败样本。
- 明显违规玩法，例如盗版资料、账号代充、群控私信、刷单控评等。
- 内容无法追溯来源，且观点无法被其他资料验证。

## 3. 标准采集流程

### 第一步：定义研究问题

不要直接搜“某领域怎么赚钱”，先把问题写清楚：

```text
研究领域：
我想解决的问题：
目标人群：
希望得到的结果：
我已有资源：
我不能碰的红线：
```

以闲鱼运营为例：

```text
研究领域：闲鱼高客单/微型服务运营
我想解决的问题：找到低风险、可测试、可标准化交付的方向
目标人群：副业新手、一个人可操作
希望得到的结果：选品清单、发布 SOP、风控清单、7 天测试计划
我已有资源：AI 工具、文档处理、设计/GIS 经验
我不能碰的红线：盗版、虚拟违规品、引流违规、医疗食品等高风险类目
```

### 第二步：搭建关键词矩阵

每个领域至少准备 6 类关键词：

| 类型 | 用法 | 示例 |
|---|---|---|
| 入门词 | 找框架 | 闲鱼运营教程、闲鱼新手流程 |
| 实操词 | 找步骤 | 闲鱼选品实操、闲鱼标题优化 |
| 案例词 | 找真实样本 | 闲鱼出单复盘、闲鱼高客单案例 |
| 数据词 | 找市场证据 | 闲鱼榜单、想要数、价格带 |
| 避坑词 | 找风险 | 闲鱼违规、闲鱼封号、闲鱼售后纠纷 |
| 工具词 | 找效率工具 | 闲鱼选品插件、商品文案生成、客服话术 |

搜索顺序建议：

1. 官方规则和平台帮助。
2. 平台内搜索和榜单。
3. B站长视频/课程转写稿。
4. 小红书/抖音短内容，用来发现近期热点。
5. 公众号/知乎长文，用来补系统化分析。
6. Google/学术/行业报告，用来验证底层逻辑。

### 第三步：保存原始资料

建议每条资料保存成一个 Markdown 文件，文件名格式：

```text
【来源平台】主题-一句话价值.md
```

正文至少包含：

```text
# 标题
原始链接：
来源平台：
作者/账号：
采集日期：
内容类型：官方规则/教程/案例/数据/工具/避坑/研究

## 原始摘录

## Codex 提炼

## 可验证证据

## 可执行动作

## 风险与疑点

## 质量评分
```

### 第四步：让 Codex 做结构化提炼

把资料放进项目后，可以让 Codex 执行这类任务：

```text
请读取项目里所有【闲鱼】相关资料，按“选品、发布、转化、风控、工具、案例”归类；
每条只保留高价值观点，删除重复内容；
对无法验证或疑似卖课/违规的内容单独列入待验证清单；
最后生成一份学习路线和 7 天行动计划。
```

或者：

```text
请对这批资料做质量评分，低于 60 分的不要进入主资料库；
把每条资料的核心观点、证据、可执行动作、风险点整理成表格。
```

### 第五步：交叉验证

高质量结论必须经过至少一种验证：

| 结论类型 | 验证方式 |
|---|---|
| 平台规则 | 官方帮助、规则页、近期处罚案例 |
| 选品机会 | 闲鱼搜索、榜单、想要数、留言时间、同款数量、价格带 |
| 运营技巧 | 至少 2 个卖家复盘或你自己的小样本测试 |
| 工具教程 | 实际安装/试用/截图验证 |
| 收益案例 | 成本、时间、订单量、售后、失败条件是否完整 |

不要把“一个博主说过”当结论。它只能算线索。

### 第六步：沉淀为行动资产

每轮搜集结束后，必须产出 4 个文件：

| 文件 | 作用 |
|---|---|
| 主题资料库 | 按模块整理全部高质量内容 |
| 结论索引 | 一页看懂核心结论和证据来源 |
| 行动清单 | 下一步可测试的动作 |
| 风险清单 | 不能碰、需验证、需观察的内容 |

### C. 7 天验证计划

| 天数 | 任务 | 输入 | 输出 | 判断标准 |
|---|---|---|---|---|
| D1 | 规则和风险排查 | 官方规则、避坑资料 | 红线清单 | 明确不能碰什么 |

## 5. 闲鱼领域的专用过滤器

优先保留：

- 官方规则、禁售规则、售后规则。
- 平台内真实数据：想要数、留言时间、同款数量、价格带、卖家等级、回复率。
- 有明确流程的卖家复盘。
- 高客单、非标、可服务化、可标准化交付的案例。
- 能转化为标题、主图、描述、客服话术、定价策略的内容。

谨慎看待：

- “无货源爆单”“日入几千”“躺赚自动化”。
- 只教补单、刷评、搬运、引流私域的内容。
- 没有成本、售后、违规风险说明的收益案例。

优先研究方向：

- 微型服务：PDF、表格、文档、图片、转写、资料整理、设计/GIS 交付。
- 高客单非标：乐器、模玩、户外、数码细分配件、圈层服饰配件。
- 信息差服务：验机、配置、代找、整理、教程、方案。

避开方向：

- 盗版课程、网盘资料、账号代充、虚拟会员。
- 高仿奢侈品、珠宝玉石泛卖、医疗药品、食品保健。
- 群控、自动私信、诱导站外交易。

## 6. 以后你可以这样指挥 Codex

### 搜集前

```text
我要研究【领域】，请帮我生成跨平台关键词矩阵、资料类型清单、质量评分标准和 7 天搜集计划。
```

### 搜集中

```text
我把今天搜集的资料放进项目了，请按 SOP 评分，低质量内容放到待验证，高质量内容进入主题资料库。
```

### 搜集后

```text
请基于项目里的资料，生成一份【领域】资料库、学习路线、行动清单、风险清单，并标注每个结论来自哪些资料。
```

### 做决策

```text
请从资料库里筛出最适合我当前资源的 5 个方向，按需求强度、竞争强度、毛利、交付难度、违规风险评分，并给出第一周测试方案。
```

## 7. 最佳协作方式

你负责“发现和保存线索”，Codex 负责“清洗、评分、归类、验证、变成行动方案”。

最理想的节奏是：

1. 每天集中搜集 30-60 分钟。
2. 把链接、截图、转写稿、笔记都放进项目。
3. 让 Codex 每天做一次资料质检和归档。
4. 每 3 天生成一次阶段总结。
5. 每 7 天输出一次可执行测试计划。

这样搜资料就不会变成无休止收藏，而会变成一个稳定的研究流水线。
