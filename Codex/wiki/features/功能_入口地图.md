# 功能：Codex 入口地图

## 一句话解释

Codex 不是单一入口，而是一组适合不同场景的工作界面：App 适合本地项目和互动式任务，CLI 适合终端工作流，IDE 适合贴近代码编辑，Cloud 适合远程仓库和并行委派，手机端适合跟进和审批桌面任务。

## 课堂讲法

先不要让学员纠结“哪个最好”，而是用任务类型来选入口：

| 入口 | 适合场景 | 不适合场景 |
|---|---|---|
| Desktop App | 本地项目、文档、互动式演示、课堂可视化 | 完全无本地依赖的远程仓库批处理 |
| CLI | 开发者终端、脚本、仓库内快速任务 | 需要大量图形界面演示的新手课堂 |
| IDE / VS Code | 解释代码、局部修改、编辑器内协作 | 非代码类资料整理 |
| Cloud | GitHub 仓库、远程长任务、PR 工作流 | 依赖本机私有环境或未提交文件 |
| Mobile | 离开电脑时查看、回复、审批桌面任务 | 在手机本地执行开发任务 |

## 课堂演示建议

- 第一屏展示 Desktop App，因为最适合课堂可视化。
- 第二步解释“同一个 Codex 能进入不同表面”，减少学生把 App 当唯一形态。
- 第三步用一张选择表让学生按任务选工具。

## 风险与边界

- UI 名称、入口位置、套餐可用性可能变化，需要官方文档或当前客户端核验。
- 手机端协同不等于任务在手机上运行。
- Cloud 任务依赖远程仓库和授权，不等于可以访问本机未保存文件。
