# Codex 课堂练习包

## 练习 1：选择入口

请为下面任务选择最适合的 Codex 入口，并说明原因。

| 任务 | 你的选择 | 原因 |
|---|---|---|
| 修改本地 Obsidian 文档 |  |  |
| 给 GitHub 仓库开 PR |  |  |
| 在 VS Code 解释一个函数 |  |  |
| 出门后审批电脑上的长任务 |  |  |
| 用终端做一次只读代码审查 |  |  |

验收标准：能说清楚本地、云端、移动端的边界。

## 练习 2：写一个好 Prompt

请把下面模糊任务改写成适合 Codex 执行的任务：

```text
帮我整理一下这个项目。
```

要求包含：

- 文件范围。
- 目标结果。
- 不要做什么。
- 验证标准。

## 练习 3：审批判断

判断下面操作是否应该放行：

| 操作 | 放行 / 拒绝 / 追问 | 原因 |
|---|---|---|
| 修改当前项目里的 Markdown |  |  |
| 访问网络下载依赖 |  |  |
| 删除项目外文件夹 |  |  |
| 读取系统环境变量 |  |  |
| 给生产数据库执行迁移 |  |  |

验收标准：能使用“必要性、风险、替代方案”三问。

## 练习 4：写最小 AGENTS.md

为一个课程知识库项目写最小 `AGENTS.md`，至少包含：

- 项目目标。
- 不允许修改的目录。
- 生成文件应该放在哪里。
- 每次维护后的验证要求。

## 延伸挑战

把你在练习 2 写出的 Prompt 交给 Codex 执行，并记录：

- Codex 读了哪些文件。
- 它改了哪些文件。
- 你如何验证结果。
- 你会如何改进 Prompt。
