# L01 Codex 入口地图

## 学习目标

- 知道 Codex 有多个使用入口。
- 能按任务选择 App、CLI、IDE、Cloud 或手机端。
- 理解本地任务、云端任务和移动端协同的边界。

## 课前准备

- 准备 Codex Desktop App 截图。
- 准备 CLI、VS Code、Cloud 的入口截图或说明。
- 如果现场无法联网，使用静态截图讲解。

## 讲授结构

| 时间 | 内容 | 形式 |
|---|---|---|
| 10 分钟 | Codex 是一组工作表面 | 讲解 |
| 15 分钟 | App / CLI / IDE / Cloud 对比 | 表格 |
| 10 分钟 | 手机端协同边界 | 案例 |
| 10 分钟 | 学员选择题 | 练习 |

## 核心概念

- Surface：不同入口适合不同工作方式。
- Local thread：依赖本机文件和环境。
- Cloud task：依赖远程仓库和云环境。
- Mobile follow-up：查看、回复、审批，不是把任务搬到手机上。

## 课堂练习

给学员 5 个任务，让他们选择入口：

- 改本地 Obsidian 文档。
- 给 GitHub 仓库开 PR。
- 在 VS Code 解释一个函数。
- 出门后审批电脑上正在跑的任务。
- 用终端执行一次只读代码审查。

## 常见坑

- 以为 Cloud 能访问本机未提交文件。
- 以为手机端能直接运行本地开发任务。
- 以为 App、CLI、IDE 是互相替代，而不是按任务选择。
