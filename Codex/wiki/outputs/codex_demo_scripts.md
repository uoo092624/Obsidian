# Codex 课堂演示脚本合集

## D01 从零完成第一个任务

### 演示目标

展示 Codex 如何读取工作目录、理解任务、修改文件、检查结果并汇报。

### 讲解词

“第一个任务不要追求酷，要追求可控。我们先让 Codex 做一个小修改，并明确告诉它怎么验证。”

### 操作步骤

1. 打开 Codex Desktop App。
2. 选择一个演示工作目录。
3. 创建或准备 `demo.md`。
4. 输入 Prompt。
5. 观察 Codex 读取文件和修改文件。
6. 检查结果。
7. 要求 Codex 总结改动。

### Prompt

```text
请先读取当前工作目录，说明你看到了哪些文件。然后只修改 demo.md：把它整理成一份面向新手的 Codex 入门清单，包含 5 个步骤和 3 个注意事项。修改后请检查 Markdown 格式，并汇报你改了什么。
```

### 失败恢复

- 没看到文件：确认项目目录。
- 想改太多：要求只改 `demo.md`。
- 触发权限：解释审批原因，再决定是否放行。

## D02 权限审批与安全边界

### 演示目标

展示 Codex 何时会请求审批，以及如何选择更安全的替代方案。

### 讲解词

“Codex 停下来问你，不是它坏了，而是当前安全边界不允许它直接做。我们判断审批时看必要性、风险和替代方案。”

### 操作步骤

1. 展示权限选择器截图。
2. 说明当前工作区范围。
3. 给 Codex 一个可能越界的请求。
4. 观察审批提示。
5. 拒绝或改写为安全替代方案。
6. 复盘为什么这样处理。

### Prompt

```text
请检查当前项目里是否有可以用于课堂演示的截图。如果你需要访问项目外目录或联网，请先说明原因、风险和替代方案，不要直接执行。
```

### 失败恢复

- 没触发审批：换成联网或项目外读取示例。
- 学员想开完全访问：回到安全边界原则。
- 审批信息看不懂：用“必要性、风险、替代方案”三问拆解。
