# 功能：安全护栏

## 一句话解释

Codex 的权限体系可以理解为三层：沙盒决定它能碰哪里，审批决定越界时谁放行，规则和项目说明决定它做事前应如何自我约束。

## 核心概念

| 概念 | 课堂解释 |
|---|---|
| Sandbox | 把 Codex 限制在一个工作范围内，减少误操作影响 |
| Approval | 当 Codex 想越过边界时暂停，让用户或审核机制判断 |
| Auto-review | 把部分审批交给审核代理判断，但不等于放开全部权限 |
| Full access | 移除很多边界，只应在明确、临时、可控的场景使用 |
| Rules / AGENTS.md | 在操作前给 Codex 更清楚的行为约束 |

## 课堂讲法

用“先默认安全，再按需放行”的顺序讲：

1. 默认从工作区写入或只读开始。
2. 出现越界需求时解释为什么要审批。
3. 展示 Auto-review 能减少打断，但不会改变沙盒边界。
4. 强调完全访问不是新手默认选项。

## 课堂演示建议

- 展示权限选择器截图。
- 演示一个需要审批的操作，例如联网或访问工作区外文件。
- 演示拒绝后如何换一个更安全的替代方案。

## 风险与边界

- 不同客户端和平台的沙盒实现可能不同。
- 配置字段可能随版本变化，课程正式发布前要核验。
- 不要把高风险组合做成“推荐配置”。
