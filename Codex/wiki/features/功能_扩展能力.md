# 功能：扩展能力

## 一句话解释

扩展能力解决的是“把一次性操作变成可复用能力”：Skills 复用流程，Plugins 打包能力，MCP 连接外部上下文和工具，Hooks 在生命周期事件中执行脚本，Automations 把任务定时或事件化。

## 概念对比

| 能力 | 课堂解释 | 适合演示 |
|---|---|---|
| Skills | 可复用任务说明、参考资料和脚本 | 把写文章、生成课件、整理资料变成技能 |
| Plugins | 打包 Skills、MCP、工具和配置 | 启用 GitHub、Browser、Docs 等能力 |
| MCP | 连接外部系统的数据和操作 | 查私有文档、项目管理、日历、仓库 |
| Hooks | 在 Codex 生命周期中执行脚本 | 提交前检查、日志、提示约束 |
| Automations | 定时或持续触发任务 | 每周生成报告、定期检查资料 |

## 课程安排建议

- 入门课只讲 Skills / Plugins 的直觉，不深入安装细节。
- 进阶课再讲 MCP、Hooks、Automations。
- 涉及外部系统授权时，必须提醒权限边界。

## 待核验

- 各入口在当前 App 中的位置。
- 插件和 Skills 的安装方式。
- MCP 和 Automation 在不同套餐/工作区中的可用性。
