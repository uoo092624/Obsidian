# D01 从零完成第一个任务

## 演示目标

展示 Codex 如何从任务描述出发，读取上下文、修改文件、验证结果并汇报。

## 环境准备

- 一个空白或示例工作目录。
- 一个简单 Markdown 文件或 HTML 文件。
- 权限使用默认或工作区写入。

## 教师讲解词

“第一个任务不要追求酷，要追求可控。我们先让 Codex 做一个小修改，并明确告诉它怎么验证。”

## 操作步骤

1. 打开 Codex Desktop App。
2. 选择或创建项目目录。
3. 放入一个简单文件，例如 `demo.md`。
4. 输入下方 prompt。
5. 观察 Codex 读文件、编辑、汇报。
6. 检查 diff 或打开文件。
7. 要求 Codex 总结它做了什么。

## 可直接使用的 Prompt

```text
请先读取当前工作目录，说明你看到了哪些文件。然后只修改 demo.md：把它整理成一份面向新手的 Codex 入门清单，包含 5 个步骤和 3 个注意事项。修改后请检查 Markdown 格式，并汇报你改了什么。
```

## 预期结果

- Codex 先读取目录。
- 只修改 `demo.md`。
- 输出清晰的修改摘要。
- 没有访问项目外文件。

## 失败恢复路径

- 如果 Codex 没看到文件：确认项目目录是否选对。
- 如果它想改太多文件：暂停并要求“只改 demo.md”。
- 如果权限卡住：解释审批原因，再决定是否放行。
