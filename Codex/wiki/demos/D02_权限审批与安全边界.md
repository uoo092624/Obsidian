# D02 权限审批与安全边界

## 演示目标

让学员看到：Codex 请求审批不是出错，而是在越过当前边界时暂停确认。

## 环境准备

- 当前项目使用默认权限或工作区写入。
- 准备一个不会造成风险的越界示例，例如请求联网查询或读取项目外文件名。

## 教师讲解词

“当 Codex 停下来问你，不是它不会做，而是当前安全边界不允许它直接做。我们要判断：这个动作对任务是否必要，风险是否可控，有没有更安全替代。”

## 操作步骤

1. 展示权限选择器截图。
2. 说明当前工作区范围。
3. 给 Codex 一个可能越界的请求。
4. 观察审批提示。
5. 拒绝或改写为安全替代方案。
6. 复盘为什么这样处理。

## 可直接使用的 Prompt

```text
请检查当前项目里是否有可以用于课堂演示的截图。如果你需要访问项目外目录或联网，请先说明原因、风险和替代方案，不要直接执行。
```

## 预期结果

- Codex 先在当前项目内检查。
- 如需越界，会说明原因并请求审批。
- 教师可以示范拒绝，并要求只使用 `attachments/`。

## 失败恢复路径

- 如果没有触发审批：说明当前请求仍在沙盒内，可换成联网或项目外读取示例。
- 如果学生想直接开完全访问：强调课堂默认不推荐。
- 如果审批信息看不懂：回到“必要性、风险、替代方案”三问。

## 截图需求

- 权限选择器。
- 审批请求或 Auto-review 拒绝示例。
