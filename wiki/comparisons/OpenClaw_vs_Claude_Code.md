# OpenClaw vs Claude Code

| 维度 | [[wiki/entities/实体_OpenClaw|OpenClaw]] | [[wiki/entities/实体_Claude_Code|Claude Code]] |
| --- | --- | --- |
| 主要场景 | 多渠道个人/团队 AI 助手 | 代码仓库内开发协作 |
| 部署方式 | 自托管为核心 | 开发工具入口为核心 |
| 生态扩展 | ClawHub / Skills / Extensions | 编码工作流和工具链集成 |
| 关注点 | 消息渠道、长期运行、任务调度 | 代码理解、编辑、验证 |

## 结论
如果核心需求是在消息渠道里长期运行一个可扩展 AI 员工，OpenClaw 更贴近；如果核心需求是在代码仓库现场完成开发任务，编码智能体更直接。
