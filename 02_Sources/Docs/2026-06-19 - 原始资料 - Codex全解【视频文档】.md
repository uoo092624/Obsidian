# Codex全解【视频文档】

## 一句话总结

这是一份 Codex 桌面端入门到进阶教程，适合加工成“Codex 入门课”的主干素材，也适合作为个人 Codex 工作流搭建的操作手册。

## 来源信息

- 来源：本地 `raw/` 目录
- 原始文件：[[Codex全解【视频文档】]]
- 本地路径：`raw/Codex全解【视频文档】.md`
- 资料类型：视频文档 / 工具教程
- 主题：Codex、AI 编程、Agent、MCP、自动化

## 原始结构

1. 准备工作：安装下载、登录方式、界面介绍
2. 基础操作：上下文管理、额度状态、模型选择
3. 本地文件读写：新建项目、权限设置、项目内任务多开
4. 命令行使用：自动审查模式、命令行执行、编辑器联动
5. 持久记忆：全局 Agents.md、项目级 AGENTS.md、自动记忆
6. 大型项目规划与落地：计划模式、引导、fork、注释、项目级 AGENT.md
7. 插件：Browser、Computer Use、Chrome、GitHub、OpenAI Developers、Netlify 等
8. Skills & CLI：推荐 Skill、Skill 创建流程、推荐 CLI
9. MCP：MCP 位置、安装方式、扩展学习资源
10. 自动化任务：自动化面板、自然语言创建自动化

## 关键观点

1. Codex 不只是聊天工具，而是可以围绕本地项目执行读写、命令行、调试、规划和自动化任务的 AI 工作台。
2. 权限模式是使用 Codex 的核心边界：默认权限、自动审查、完全访问分别适合不同风险级别的任务。
3. Codex 的价值来自“项目上下文 + 工具调用 + 持久记忆 + 工作流复用”，而不只是一次性问答。
4. 对复杂项目，应先用计划模式澄清目标、步骤和验收标准，再进入执行。
5. AGENTS.md 是把个人偏好、项目规则和工作流沉淀下来的关键文件。
6. 插件、Skills、CLI、MCP 是 Codex 扩展能力的四条路径，分别面向浏览器、专用技能、命令行工具和外部服务连接。
7. 自动化任务适合把周期性内容生产、资料整理、项目推荐等流程固定下来。

## 值得摘录

> 默认权限：仅能读写所选文件夹内全部文件，若需访问文件夹外内容，Codex 要单独申请权限。

> 自动审查：AI 自动检查操作是否存在风险，判定有风险就请示，无风险则直接执行操作。

> 针对有一定复杂度的项目，或者我们自己都还没想清楚的项目，可开启计划模式。

## 我的理解

这份资料的价值不在于某一个单点功能，而在于它给出了 Codex 的完整使用版图：

- 新手入口：安装、登录、界面、模型和额度
- 工程入口：本地文件读写、命令行、权限管理
- 长期使用入口：AGENTS.md、自动记忆、项目规则
- 项目管理入口：计划模式、fork、注释反馈、项目总结
- 扩展入口：插件、Skills、CLI、MCP、自动化

如果用于课程设计，可以把它拆成“基础操作”“项目协作”“长期记忆”“扩展生态”“自动化工作流”五个模块。
